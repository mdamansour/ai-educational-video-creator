0. CONFIRMED DESIGN DECISIONS
Decision	Choice
Karaoke timing	Time-based (estimatedDuration / sentenceCount)
Subtitles	Exported as .srt sidecar (burn-in optional)
BGM	5 bundled royalty-free tracks + custom file import
Project format	Workspace folder (human-readable, debuggable)
Logging	Structured UI log panel + JSONL session file
MC Editor	Natively integrated as a core interactive loop

┌──────────────────────────────────────────────────────────────────────────┐
│ STEP 1  │ Topic + Audience Level + Video Format / Resolution             │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 2  │ AI Research          [ResearchAgent → Gemini]                  │
│         │ Concepts · Definitions · LaTeX · Analogies · Misconceptions    │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 3  │ Scenario Design      [VisualDirectorAgent → Gemini]            │
│         │ Per-scene visual type · ScenePlanJSON generated                │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 4  │ Script Review & Edit [PedagogyAgent → Gemini]                  │
│         │ hook→build-up→aha→payoff arc · User edits narration text       │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 5  │ 🎙 Voice Recording    [MANUAL — Karaoke Studio]                │
│         │ Scene-by-scene · Script highlighted in sync · VU meter         │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 6  │ 🎨 Motion Canvas Editor [INTERACTIVE — Core Loop]             │
│         │ AI generates .tsx → MC preview → User edits → AI reads back   │
│         │ Monaco editor · Live MC preview · SceneReadBack feedback loop  │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 7  │ Audio Mixing          [Auto: Mixer + Ducker]                   │
│         │ Narration + BGM (bundled or custom) + SFX · Auto-ducking       │
├──────────────────────────────────────────────────────────────────────────┤
│ STEP 8  │ Render                [RenderAgent → MC headless]              │
│         │ project.ts stitched · MC renders frames → video               │
├──────────────────────────────────────────────────────────────────────────┤

│         │ .mp4 / .webm · .srt sidecar · Format-aware resolution         │

4. CORE SUBSYSTEM DESIGNS
4.1 Logging System

Event Source (any module)
        │
        ▼
Logger.log(level, source, message, metadata?)
        │
        ├──→ Async in-memory queue (non-blocking)
        │           │
        │           ├──→ IPC emit → renderer → logStore (Zustand) → LogPanel UI
        │           │
        │           └──→ Append to <workspace>/.logs/session_<ISO>.jsonl
        │
        └── (on ERROR/FATAL) → also appended to lastErrors[] for "Copy Last Error"
LogEntry schema:

typescript

interface LogEntry {
  id: string;                    // uuid
  timestamp: string;             // ISO-8601
  level: "DEBUG" | "INFO" | "WARN" | "ERROR" | "FATAL";
  source: string;                // e.g. "ResearchAgent", "MCDevServer", "Recorder"
  message: string;
  metadata?: {
    prompt?: string;             // Full prompt sent to Gemini (on failure)
    rawResponse?: string;        // Raw LLM response (on failure)
    zodErrors?: string[];        // Validation errors
    filePath?: string;
    durationMs?: number;
    attemptNumber?: number;
    [key: string]: unknown;
  };
}

     │                                   ▼

     │                                   │
     │                                   ▼

     └── Types instruction in AIInstructionBar
               │
               ▼
         SceneReadBackAgent reads CURRENT .tsx from disk
         (always ground truth, never cached)
               │
               ▼
         CodeGenAgent generates a PATCH (not a full rewrite)
               │
               ▼
         CodeReviewAgent: tsc --noEmit passes?
               │
         Yes → file written → hot-reload → user sees result
         No  → retry with compiler error → up to 3×
               │
               ▼
         [User reviews] → loop back to top
Key Invariant (FR-E11):

The AI NEVER operates on a cached string. Before any AI action on a scene, the system reads the .tsx file from disk. This is the single source of truth.

4.3 LLM Prompt Engineering Strategy
Mandatory 8-Section Prompt Template

1. [SYSTEM PERSONA]
   "You are a Motion Canvas v3 animation expert. You write only
    syntactically correct TypeScript using the Motion Canvas API.
    You do NOT invent API methods. You do NOT add explanations."
2. [TASK — one imperative sentence]
   "Generate a Motion Canvas scene for the following input."
3. [INPUT DATA — clearly labeled JSON]
   "=== SCENE PLAN ===\n<json>"
4. [OUTPUT CONTRACT — exact schema inline]
   "Return ONLY a JSON object matching this exact structure:
    { sceneCode: string, components: string[] }"
5. [FEW-SHOT EXAMPLES — 1 or 2 complete pairs]
   "=== EXAMPLE INPUT ===\n...\n=== EXAMPLE OUTPUT ===\n..."
6. [NEGATIVE EXAMPLES]
   "=== DO NOT DO THIS ===
    Bad: using `scene.add()` — this API does not exist in MC v3.
    Bad: adding markdown fences around the JSON output.
    Bad: adding any key not in the output contract."
7. [CHAIN-OF-THOUGHT CUE]
   "Think step by step about which MC components match the visual type,
    then write the code, then verify imports match components used."
8. [FINAL INSTRUCTION]
   "Return ONLY valid JSON. No markdown. No extra keys. No commentary."
Validation + Retry Loop

Gemini response
      │
      ▼
  safeJsonParse()   ──── fail ──→  inject parse error → retry (max 3)
      │
      ▼
  zodSchema.parse() ──── fail ──→  inject Zod errors  → retry (max 3)
      │
      ▼
  domainValidate()  ──── fail ──→  inject domain rule → retry (max 3)
      │
      ▼
  AgentOutput ✓ (fully typed, guaranteed valid)
              │
              └──→ full prompt + response logged at DEBUG level always
                   on failure: logged at ERROR with all metadata
Per-Agent Hallucination Guards
Agent	Risk	Guard
ResearchAgent	Fabricating facts/formulas	"Do NOT invent. Flag uncertainty with 'commonly believed'."
PedagogyAgent	Missing arc phases / zero durations	All 4 phases required in Zod; duration >= 5 enforced
VisualDirectorAgent	Non-enum visual types	Full enum embedded verbatim in prompt; any deviation → retry
CodeGenAgent	Invalid MC v3 API calls	MC v3 API reference + 2 full examples in few-shot; forbidden method list in negative examples
CodeReviewAgent	"Fixing" by deleting code	Instructed to produce minimal diffs; never remove entire functions
SceneReadBackAgent	Misunderstanding user edits	Reads raw .tsx → produces structured JSON; user confirms before write
5. KEY DATA CONTRACTS
ScriptJSON
typescript

interface ScriptJSON {
  topic: string;
  audienceLevel: "middle" | "high_school" | "university" | "graduate";
  videoFormat: FormatPreset;
  totalEstimatedDuration: number;      // seconds
  scenes: ScriptScene[];
}
interface ScriptScene {
  id: string;                          // "scene_01", ...
  arcPhase: "hook" | "buildup" | "aha" | "payoff";
  estimatedDuration: number;           // seconds, ≥ 5
  narration: string;                   // full spoken text
  sentences: string[];                 // split narration for karaoke
  onScreenText?: string;
  latexFormula?: string;
  keywords: string[];
  locked?: boolean;                    // user-locked in MC editor
}
ScenePlanJSON
typescript

interface ScenePlanJSON {
  videoFormat: VideoFormat;
  scenes: ScenePlan[];
}
interface ScenePlan {
  sceneId: string;
  visualType: VisualType;              // strict enum
  sourceOfTruth: "ai_generated" | "user_edited" | "ai_readback";
  lastReadFromDisk?: string;           // ISO timestamp of last SceneReadBackAgent run
  motionCanvasHints: {
    components: MCComponent[];         // strict enum
    animationStyle: AnimationStyle;
    colorPalette: string[];
    cameraMovement?: CameraMovement;
  };
  estimatedDuration: number;
  scriptRef: string;
  locked: boolean;
}
